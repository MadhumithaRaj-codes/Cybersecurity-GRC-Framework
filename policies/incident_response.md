# Incident Response Policy

## Objective
To provide a structured approach to handling security incidents.

## Phases
1. Preparation
2. Identification
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

## Roles
- Incident Response Team (IRT)
- Security Analyst
- IT Administrator

## Communication Plan
- Internal reporting within 1 hour
- External reporting based on severity

## Example Incidents
- Phishing attack
- Malware infection
- Data breach
